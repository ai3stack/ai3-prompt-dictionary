## How attribution works

- **Prompt edits**: every prompt's commit history is the canonical record of who improved it. We do not stamp contributor names inside the JSON itself (would balloon diffs and complicate review).
- **Bulk batches**: if you add or revise ≥50 prompts in a session, a single line under "All Contributors" is enough — no need to enumerate IDs.
- **Code contributions**: same — git log is canonical, this file is the human-readable summary.

## Recognition beyond this file

- Twice a year (May / November), the BDFL posts a **contributor recap** in [Discussions](https://github.com/ai3stack/ai-prompt-dictionary/discussions) highlighting standout PRs and graduating Reviewers / Committers.
- Top-3 contributors of the year by merged-PR count get featured on the consumer website's `/about` page (with permission).
